Union-Find Advanced

class DSU {
    int[] parent, rank;
    DSU(int n) {
        parent = new int[n];
        rank = new int[n];
        for (int i = 0; i < n; i++) parent[i] = i;
    }
    int find(int x) {
        if (parent[x] != x) parent[x] = find(parent[x]);
        return parent[x];
    }
    boolean union(int a, int b) {
        int ra = find(a), rb = find(b);
        if (ra == rb) return false;
        if (rank[ra] < rank[rb]) parent[ra] = rb;
        else if (rank[ra] > rank[rb]) parent[rb] = ra;
        else { parent[rb] = ra; rank[ra]++; }
        return true;
    }
}

Object-Oriented Design

  • Entities명사 후보를 나열하고 책임이 큰 객체부터 쪼갭니다.
  • APIpublic 메서드는 사용 사례 중심, 내부 상태 변경은 작게 캡슐화합니다.
  • Extensibility새 타입이 늘어날 축에는 interface/strategy를 둡니다.
  • Tradeoff단순 구현과 과설계 사이에서 현재 요구사항을 기준으로 방어합니다.

System Design

  • Scope읽기/쓰기 비율, QPS, 데이터 크기, latency target을 먼저 고정합니다.
  • Shapeclient, API, service, storage, cache, queue, worker를 박스로 나눕니다.
  • Scalestateless service, partition key, replication, cache invalidation을 설명합니다.
  • Failureretry, idempotency key, circuit breaker, backpressure를 언급합니다.

Database

  • Indexwhere/order by/join column에 맞추되 write cost와 cardinality를 같이 봅니다.
  • Txatomicity, isolation anomaly, lock 범위를 예시로 설명합니다.
  • Schema정규화로 무결성 확보, 조회 병목에는 선택적 비정규화와 materialized view를 고려합니다.
  • NoSQLaccess pattern이 고정되고 scale-out이 중요할 때 후보로 둡니다.

Threads and Locks

  • Race공유 상태에 read-modify-write가 있으면 동기화 필요성을 먼저 짚습니다.
  • Deadlocklock 획득 순서 통일, timeout, lock scope 축소로 방지합니다.
  • Visibilityvolatile은 가시성, synchronized/Lock은 상호 배제와 happens-before를 제공합니다.
  • Practiceproducer-consumer, bounded queue, thread-safe cache를 말로 설계합니다.

Testing

  • Unit정상, 경계, 실패, 랜덤, 회귀 케이스를 분리합니다.
  • Property정렬 결과는 길이/원소 multiset/순서 조건을 만족해야 합니다.
  • Mutation비교 연산자, off-by-one, null 처리 변경에 깨지는 테스트를 둡니다.
  • Debug작은 입력을 손으로 trace하고 invariant가 깨지는 줄을 찾습니다.
